## Design Context

### Users
Primary user is Aram using this as a personal high-throughput job-search control surface. Context is mobile-first and fast triage: review daily job briefs, decide apply vs skip quickly, and move dozens of roles per day without friction.

### Brand Personality
Direct, calm, utilitarian. High-signal, no-fluff, operator-grade. The interface should feel trustworthy, fast, and slightly premium without becoming marketing-y.

### Aesthetic Direction
A compact decision dashboard, not a startup landing page. Prioritize legibility, hierarchy, and speed over decorative flourish. Dark mode is acceptable, but avoid generic AI-slop cues like neon gradients, glass cards, glowing accents, and templated hero layouts.

### Design Principles
1. Optimize for fast triage and low cognitive load.
2. Make the next action obvious within 2 seconds.
3. Surface only information that changes the apply/skip decision.
4. Mobile usability matters as much as desktop readability.
5. Prefer crisp structure and feedback over decorative styling.
