## Design Context

### Users
Primary user is Aram, operating under cognitive fatigue during a live job search. He needs a single decision surface that compresses discovery, triage, and application preparation into one fast feed. The core job to be done is: open one screen, understand what is new / worth attention / already prepared, then take the next obvious action with minimal mental overhead.

### Brand Personality
Calm, sharp, operational.
The interface should feel high-signal, trustworthy, and anti-clutter. Emotionally it should create confidence and relief: no duplicate noise, no fake activity, no hidden state. It should feel like a product operator’s control surface, not a report archive or ATS.

### Aesthetic Direction
Dark mode first. Editorial-product hybrid rather than generic dashboard. Dense but breathable. Strong hierarchy, compact chips, premium typography, and minimal decorative noise. Avoid AI-slop dashboard clichés, card spam, and mirrored grid repetition. The primary pattern is a vertical feed with section breaks, not a spreadsheet and not a kanban wall.

### Design Principles
1. Feed first, not archive first: the default surface is a living operational timeline of decisions and statuses.
2. Compress to decisions: every card must help the user decide pursue / later / skip quickly.
3. Preserve trust: hidden or rejected role families must not quietly resurface.
4. Separate truth states clearly: new today, best current targets, preparing, ready, blocked, submitted must never blur together.
5. Scan speed beats completeness: show the smallest useful amount of copy by default, reveal detail on demand.
